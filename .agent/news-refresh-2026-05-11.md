# News Refresh 2026-05-11 ExecPlan

## 1. Objective

Add source-backed AI tools news items published after the latest existing AiPedia news entries, improving freshness, trust, organic coverage, and decision support without creating thin or unverified pages.

## 2. Current state

News content lives in `src/content/news/*.md` and is rendered by `src/pages/news/[slug].astro` plus the `/news/` hub at `src/pages/news/index.astro`. RSS is generated by `src/pages/news/rss.xml.ts`. News OG and thumbnail images are generated by `scripts/generate-og-news.mjs` into `public/og/news/`, `public/og/news/light/`, `public/og/news/thumbs/`, and `public/og/news/thumbs/light/`.

The newest existing news files are dated `2026-05-08`. `PAGE_REFRESH_LEDGER.md` intentionally excludes individual news articles but tracks `/news/`, `/news/rss.xml`, homepage, sitemap/LLM surfaces, and other parent pages.

## 3. Target state

Add only material, currently verified news items from May 8-11, 2026 that affect AI tool buying decisions, tool positioning, or category direction. Each article must include frontmatter sources, last verified date, related tools/categories, and concise decision-oriented analysis. Generate all required news images.

## 4. Scope

Included: new `src/content/news` articles, generated OG/thumb assets, ledger regeneration, news xref checks, content guards, and build validation.

Excluded: new tool records, tool pricing changes, affiliate CTA changes, and speculative articles without current source support.

Affected top-layer surfaces: `/news/`, `/news/rss.xml`, `/`, `/tools/`, `/categories/`, `/compare/`, `/guides/`, `/trends/`, `/workflows/`, `/answers/`, `/explore/`, `/llms.txt`, `/llms-full.txt`, sitemap surfaces through generated build.

## 5. Files likely affected

- `src/content/news/*.md`
- `public/og/news/**`
- `PAGE_REFRESH_LEDGER.md`
- Possibly generated/crawl surfaces if ledger scripts update them

## 6. Data model impact

No schema changes. New news entries use the existing `news` collection fields: `slug`, `title`, `date`, `severity`, `summary`, `categories`, `affects`, `related_tools`, `sources`, `last_updated`, and `last_verified`.

## 7. SEO impact

Each article receives a unique title, canonical via existing template, source-backed body copy, article schema through existing layout, and inclusion in `/news/`, RSS, sitemap, search, and LLM surfaces via content collection.

## 8. Conversion impact

No direct affiliate CTA edits. Articles can influence downstream tool pages through `affects` and `related_tools` links.

## 9. Mobile UX impact

Existing news article and card templates are mobile-first. Generated thumbnails must exist so mobile listing cards and article hero areas do not show missing assets.

## 10. Implementation steps

1. Verify latest existing news date and article conventions.
2. Research current May 2026 AI/tool news with queries including `May 2026`.
3. Select only material stories not already represented in the repo.
4. Add concise but high-signal article files.
5. Generate news OG/thumb images.
6. Regenerate page refresh ledger and inspect diffs.
7. Run news/content checks and build.

## 11. Verification commands

- `npm exec --yes --package=node@24 -- node scripts/generate-og-news.mjs`
- `npm run ledger:pages`
- `npm run check:news`
- `npm run guard:check`
- `npm run build:fast`

## 12. Acceptance criteria

New articles are source-backed, dated/verified `2026-05-11`, have generated image assets in all required news image folders, pass content/news checks, build successfully, and parent/news/crawl surfaces are current in the ledger.

## 13. Risks and mitigations

Risk: Search results include stale or speculative reports. Mitigation: prefer primary or named publication sources and avoid unsupported claims.

Risk: Adding too many thin posts weakens news quality. Mitigation: ship only material items with clear buyer/operator implications.

Risk: Existing working tree is already dirty. Mitigation: avoid reverting unrelated changes and report only files touched for this task.
