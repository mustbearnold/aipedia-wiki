# ExecPlan: Workflow Freshness And Trust Cleanup

## 1. Objective
Refresh or noindex stale `/workflows/` pages so AiPedia workflow tutorials stop carrying April 2026 pricing/model/version claims, unsupported time-saved promises, and stale top-layer summaries. The goal is better trust, SEO quality, mobile buyer clarity, and conversion paths into current tool and guide pages.

## 2. Current state
- Workflow pages are Markdown records in `src/content/workflows/*.md`.
- Routes are generated by `src/pages/workflows/[slug].astro` and rendered through `ArticlePlusLayout`.
- The parent hub is `src/pages/workflows/index.astro`.
- The workflow schema in `src/content.config.ts` currently supports `type`, `slug`, `title`, `seo_title`, `meta_description`, `description`, `stack`, `tools_mentioned`, `author`, `last_updated`, `last_verified`, and `update_frequency`, with passthrough fields allowed.
- The workflow route currently emits every workflow entry and does not filter `noindex` pages out of static paths.
- All seven workflow files contain stale April 2026 language or brittle pricing/model claims:
  - `src/content/workflows/youtube-content-stack.md`
  - `src/content/workflows/podcast-automation-stack.md`
  - `src/content/workflows/newsletter-stack.md`
  - `src/content/workflows/research-assistant-stack.md`
  - `src/content/workflows/agentic-coding-workflow.md`
  - `src/content/workflows/design-agency-replacement.md`
  - `src/content/workflows/micro-saas-weekend-build.md`

## 3. Target state
- High-value workflow pages are refreshed with current sources as of 2026-05-09.
- Weak/overlapping workflows are noindexed or merged into stronger guides if a full refresh is not justified.
- `/workflows/` clearly surfaces only current, decision-useful workflow tutorials.
- Updated pages avoid fake precision around exact time savings, freelance replacement claims, exact monthly totals, or unsupported model/version names.
- Mobile pages lead with a practical stack decision, buying order, avoid-if guidance, and links to stronger money guides.

## 4. Scope
Included:
- First checkpoint: `youtube-content-stack` and `/workflows/`.
- Add workflow-level `noindex` filtering if we choose to archive weak workflow pages.
- Maintain internal links to related guides and tools.
- Update this ExecPlan and any triage notes.

Excluded for this checkpoint:
- Full template rebuild of `ArticlePlusLayout`.
- Full refresh of all seven workflows in one commit.
- New tool records unless a workflow refresh genuinely needs a missing tool and a proper logo can be sourced.

Affected top-layer pages and surfaces:
- `src/pages/workflows/index.astro`
- Sitemap/indexability through `src/pages/workflows/[slug].astro` if noindex filtering is added.
- Related guide pages only if their summaries are changed; otherwise link to them without modifying.

## 6. Data model impact
No required schema migration. If needed, workflow records may use passthrough `noindex: true`, but the route must explicitly honor it before relying on noindex behavior.

## 7. SEO impact
Refreshing high-value workflows should improve freshness, internal linking, and trust. Noindexing weak pages should reduce crawl-quality drag from duplicated or stale stack pages.

## 8. Conversion impact
Workflow pages should route readers to relevant tool pages, buyer guides, and comparison pages with honest purchase sequencing. Avoid exact savings claims unless sourced.

## 9. Mobile UX impact
Updated workflows must be checked at 360, 390, 430, 768, and 1024. No wide table should be the primary mobile experience. If tables remain, they must be secondary and not cause horizontal overflow.

## 10. Implementation steps
1. Inspect `youtube-content-stack` and parent `/workflows/` rendering.
2. Verify current volatile facts with official/vendor sources before editing.
3. Refresh or noindex `youtube-content-stack` based on source-backed value.
4. Update `/workflows/` parent copy/cards to match the changed child page.
5. Add triage notes for the remaining workflow pages.
6. Run `npm run check:content`, `npm run ship:check`, and focused Playwright mobile smoke checks.
7. Commit and push the checkpoint only after checks pass.

## 11. Verification commands
- `npm run check:content`
- `npm run ship:check`
- `git diff --check`
- Focused Playwright smoke for `/workflows/` and the changed workflow route at 360, 390, 430, 768, and 1024.

## 12. Acceptance criteria
- Changed workflow content uses current 2026-05-09 source verification.
- No unsupported April pricing/model/version claims remain in the changed workflow page.
- Parent `/workflows/` is current and aligned.
- Checks and focused mobile smoke pass.
- Remaining workflow risks are documented with exact next actions.

## 13. Risks and mitigations
- Risk: exact stack prices go stale quickly. Mitigation: use price bands and plan caveats unless official pricing supports exact statements.
- Risk: overlap with `/guides/ai-content-creator-stack/`. Mitigation: make the workflow a step-by-step operating tutorial or merge/noindex if it cannot add unique value.
- Risk: noindex fields do nothing if route ignores them. Mitigation: update route behavior before relying on noindex.
