# ExecPlan: Workflow Freshness and Buyer-Flow Checkpoint

## 1. Objective
Refresh stale `/workflows/` pages that carry April 2026 product, pricing, or model claims so AiPedia workflow tutorials become current, source-backed, mobile-safe buyer paths instead of brittle fixed-cost recipes.

## 2. Current state
- Workflow pages live in `src/content/workflows/*.md`.
- Workflow routes are generated by `src/pages/workflows/[slug].astro` through `ArticlePlusLayout`.
- The parent workflow hub is `src/pages/workflows/index.astro`.
- `agentic-coding-workflow` and `micro-saas-weekend-build` both had April 2026 verification dates and unsupported fixed-cost/version/time-saved claims.

## 3. Target state
- Kept workflow pages explain a real operating pattern, buying order, failure points, and human checkpoints.
- Volatile tool facts are verified against current vendor sources as of 2026-05-09.
- `/workflows/` shows freshness signals and does not bury newly refreshed pages.
- Affected category surfaces link to refreshed workflows when they support buyer intent.

## 4. Scope
Included:
- Refresh the first high-impact workflow page: `/workflows/agentic-coding-workflow/`.
- Update `/workflows/` parent hub freshness/sorting.
- Update affected category parent surface: AI Coding.

Excluded:
- Full rebuild of every workflow page in one pass.
- Adding non-AI infrastructure tools like Supabase, Cloudflare, Stripe, or Vercel as catalog records without a separate logo/data pass.

## 5. Files likely affected
- `src/content/workflows/agentic-coding-workflow.md`
- `src/pages/workflows/index.astro`
- `src/content/categories/ai-coding.md`
- `.agent/WORKFLOW_FRESHNESS_2026_05_09_EXECPLAN.md`

## 6. Data model impact
No schema migration. Workflow frontmatter continues to use existing `stack`, `tools_mentioned`, `last_updated`, and `last_verified` fields.

## 7. SEO impact
Improve freshness, title/meta alignment, internal links, source-backed trust, and reduce crawl-quality risk from stale unsupported version/cost claims.

## 8. Conversion impact
Route users to relevant tool pages for Cursor, Claude Code, GitHub Copilot, and Codex while keeping warnings about cost control and human review.

## 9. Mobile UX impact
Workflow content should be readable as stacked sections at 360, 390, 430, 768, and desktop widths; wide tables must not create horizontal overflow.

## 10. Implementation steps
1. Inspect stale workflow pages and choose the highest-value next page.
2. Verify current volatile facts for the selected tools.
3. Rewrite stale claims into durable buyer/workflow guidance.
4. Update `/workflows/` parent hub and affected category surface.
5. Run content, build, mobile smoke, and ship checks.
6. Commit and push a clean checkpoint.

## 11. Verification commands
- `npm run check:content`
- `git diff --check`
- Mobile smoke at 360, 390, 430, 768, 1024 for `/workflows/agentic-coding-workflow/`, `/workflows/`, and `/categories/ai-coding/`
- `npm run ship:check`

## 12. Acceptance criteria
- `agentic-coding-workflow` has May 9 verification and current official sources.
- Fixed stale versions, fake throughput, and fixed monthly totals are removed or caveated.
- `/workflows/` lists current verification dates and surfaces refreshed workflows.
- AI Coding parent links to the refreshed workflow.
- Checks pass and the checkpoint is pushed.

## 13. Risks and mitigations
- Risk: replacing a precise workflow with vague advice. Mitigation: keep concrete handoff steps, failure points, and tool-by-job table.
- Risk: introducing new unsupported cost claims. Mitigation: cite current sources and frame costs as usage-sensitive.
- Risk: parent hub drift. Mitigation: update `/workflows/` and AI Coding before finishing.
