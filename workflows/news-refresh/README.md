# Daily News Refresh Workflow

Daily AI and AI-tools news is AiPedia's highest-priority freshness loop. Run it before affiliate expansion, broad page refresh, tool refresh, visual polish, or comparison work unless production is broken.

## Objective

Keep the public news surface visibly current so readers do not land on AiPedia and conclude the site is stale.

Each daily pass should answer two questions:

- What changed in frontier AI, labs, policy, infrastructure, model access, safety, copyright, or platform distribution?
- What changed for AI tools buyers: launches, pricing, deprecations, model access, reliability, enterprise controls, tool governance, affiliate-relevant commercial changes, or buyer-risk signals?

Publish both broad AI news and AI-tools news only when current sources support both. If one lane has no source-backed story worth publishing, record that explicitly in the work log.

## Source Discovery

Use current-date web checks. Every search for volatile news must include the current month and year.

Prioritize primary sources first:

- Official company newsrooms, blogs, release notes, changelogs, docs, pricing pages, status pages, and SEC or regulator pages.
- Credible secondary reporting only when the primary source is absent, incomplete, or the story is about reporting itself.
- Social posts only as supporting evidence unless they are official account announcements and the linked source confirms the claim.

Do not publish rumor-only stories, stale roundup filler, or thin "AI news today" pages without buyer relevance.

## Story Bar

Prefer 2 to 4 source-backed stories per day. At least one should be AI-tools, tool-control, model-access, pricing, reliability, or buyer-risk oriented when current sources support it.

Each story needs:

- A clear why-it-matters angle for AI buyers, builders, or tool evaluators.
- Source-backed claims with current source dates.
- Concrete impact on tools, categories, comparisons, trends, or buyer decisions.
- Internal links to affected tools, categories, guides, comparisons, trends, or prior news.
- Plain editorial-analysis voice, no vendor hype.

## Surfaces To Update

After adding or refreshing news, inspect and update affected surfaces:

- `/news/`
- `/`
- Affected tool pages or category hubs when the story changes buyer guidance.
- `/trends/model-availability-churn/` when model availability, pricing, access, or frontier-model status changes.
- RSS/feed, sitemap, LLM surfaces, search/explore data, and `PAGE_REFRESH_LEDGER.md` where generated by the site.
- OG and thumbnail assets for new news pages.

## Verification

Use the smallest valid set during drafting, then the full rendered set before publishing.

Minimum final commands for a rendered daily news pass:

```bash
npm run ledger:pages
npm run ledger:pages:check
node scripts/guard-em-dashes.mjs
npm run check:news
node scripts/audit-news-rendering.mjs --json
npm run check:links
npm run typecheck
npm run build:fast
npm run qa:route -- --site-dir dist-fast/client --concurrency 6 --widths 319,360,390,430,768,1024,1366 --route /news/ --route /
git diff --check
```

Add exact new article routes and any affected parent/category/tool/trend routes to route QA. If `/search/` or `/explore/` is materially affected, include them with `--allow-noindex`.

## Closeout

Record:

- Stories published and skipped lanes.
- Sources checked and any blocked/current-source uncertainty.
- Parent/top-layer surfaces updated.
- Verification commands and failures fixed.
- Residual follow-ups for affected tool pages, categories, comparisons, or trends.
