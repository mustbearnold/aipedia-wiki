# ServiceNow Build Agent reaches Studio and major AI coding tools

ServiceNow announced on May 6, 2026, that Build Agent is generally available in ServiceNow Studio and that its core skills now extend into Cursor, Windsurf, Claude Code, and GitHub Copilot.

The product pitch is governed app development. Developers can work from ServiceNow Studio, a preferred IDE, or an AI coding agent while Build Agent retains ServiceNow platform context, including data models, configurations, policies, workflows, and deployment controls.

ServiceNow also says Build Agent can operate across application scopes, including out-of-the-box applications rather than only custom apps. App Engine Management Center is positioned as the governance layer for approvals, release management, and lifecycle controls, while Build Agent adds a self-healing test loop and customer-specific instructions.

## Why this matters

AI coding tools are increasingly good at generating code, but enterprise apps still need governance, audit trails, release processes, security checks, and platform-specific knowledge.

ServiceNow is trying to meet developers in the coding tools they already use while keeping the resulting apps inside the ServiceNow AI Platform governance model.

## Buyer take

If your ServiceNow backlog is constrained by specialist developer capacity, Build Agent is worth a controlled pilot. Test it on an internal workflow with clear acceptance criteria, release gates, and rollback.

For teams using Cursor, Windsurf, Claude Code, or GitHub Copilot, treat the ServiceNow integration as a privileged platform connector. Review permissions, generated changes, and deployment approvals before giving it broad access.

## What is still unclear

Some ecosystem pieces are staged. The announcement lists Build Agent in Studio as generally available, Build Agent Skills for agentic development tools as available since April 2026, and MCP Client ecosystem integrations as expected in the second quarter of 2026.
