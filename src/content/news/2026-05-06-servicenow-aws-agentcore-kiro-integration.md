# ServiceNow and AWS link AI Control Tower, Bedrock AgentCore, and Kiro

ServiceNow and AWS announced a platform expansion at Knowledge 2026 on May 6, 2026, alongside a milestone that ServiceNow transactions through AWS Marketplace have surpassed $1 billion.

The product news centers on a shared governance architecture using ServiceNow AI Control Tower and Amazon Bedrock AgentCore. ServiceNow says the integration is meant to let enterprises deploy agents on AWS infrastructure while governing and wiring the work into ServiceNow workflows.

The release also mentions new AI agent integrations for enterprise security, IT operations, and telecommunications, plus a native developer integration that lets teams build and deploy ServiceNow applications directly from Kiro, AWS's agentic IDE.

## Why this matters

Enterprise AI stacks are becoming multi-platform by default. A model or agent runtime may live in AWS, but the workflow, approval, ticket, or operational action often lives in ServiceNow.

The Kiro link matters because spec-driven coding agents are moving closer to governed enterprise platforms. Developers can start from a Kiro workflow while targeting ServiceNow applications and governance constraints.

## Buyer take

Mutual AWS and ServiceNow customers should evaluate whether AI Control Tower plus Bedrock AgentCore reduces duplicate governance work or simply adds another integration layer.

Kiro users should verify what the ServiceNow integration can actually generate, deploy, and audit before using it for production workflows.

## What is still unclear

The announcement bundles marketplace momentum, governance architecture, agent integrations, and Kiro developer workflow. Each piece may have different availability, entitlements, and implementation requirements.
