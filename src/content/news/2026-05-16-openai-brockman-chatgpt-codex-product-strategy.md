# OpenAI puts Greg Brockman over product strategy as ChatGPT and Codex converge

OpenAI has made Greg Brockman's product role official. WIRED reported on **May 15, 2026** that Brockman now leads OpenAI's product strategy in addition to his infrastructure work, and OpenAI confirmed the move. TechCrunch followed on **May 16** with OpenAI's note that the change was made while Fidji Simo remains on medical leave.

The important buyer signal is not just the org chart. OpenAI is folding **ChatGPT**, **Codex**, and the developer-facing API into one core product team. WIRED also reported that Brockman described a plan to merge ChatGPT and Codex into one unified experience.

## What changed

OpenAI is treating agentic work as the center of the product line rather than a separate coding lane. Codex already appears across ChatGPT, the desktop app, CLI, IDE extensions, and mobile control surfaces. The reorg makes that convergence explicit.

For ChatGPT users, this means more coding-agent capability is likely to show up inside the main assistant. For Codex users, it means Codex is increasingly tied to ChatGPT's consumer and enterprise packaging rather than becoming a standalone developer-only subscription.

OpenAI's current pricing page also reflects that direction: Codex access appears inside Free, Go, Plus, Pro, Business, and Enterprise plan comparisons, while Business Codex is listed as a pay-as-you-go development-focused plan.

## Why this matters

AI tool buyers have had to choose between general assistants and specialized coding agents. OpenAI is trying to collapse that choice by turning ChatGPT into the front door for agentic coding, document work, browser work, API workflows, and enterprise automation.

That raises the stakes for procurement. If a team adopts ChatGPT for general work, it may also be adopting its coding-agent roadmap by default. If a team already uses Cursor, Claude Code, GitHub Copilot, or Devin, this is a reason to revisit where Codex fits in the stack instead of treating it as a side product.

## Buyer take

For individual users, the practical call remains simple: use ChatGPT Plus if you want broad assistant capability and occasional Codex work; move to Pro only if the higher agent limits matter.

For engineering teams, compare Codex against Claude Code and Cursor on the actual workflow: repository context, reviewability, sandboxing, and handoff quality. The reorg improves OpenAI's product focus, but it does not remove the need to test failure recovery and permission boundaries.

## What to watch next

Watch for a unified ChatGPT-Codex desktop experience, stronger cross-surface memory, and clearer Business Codex pricing. Also watch whether OpenAI reduces product sprawl after folding ChatGPT, Codex, and the API into one core team.
