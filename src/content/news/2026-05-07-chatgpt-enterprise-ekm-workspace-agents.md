# ChatGPT Workspace Agents reach Enterprise workspaces with EKM

OpenAI's Enterprise and Edu release notes say ChatGPT Workspace Agents are now available to eligible ChatGPT Enterprise workspaces with Enterprise Key Management.

Workspace Agents let organizations build repeatable agents for business workflows across connected apps. OpenAI says eligible EKM workspaces can create agents, connect supported tools and apps, add skills, files, and custom MCP servers, schedule recurring runs, use agents in connected Slack channels, and view version history and analytics.

The release note says workspace agents remain off by default. Admins can enable agent building, publishing, and Slack usage through admin controls.

## Why this matters

This is a buyer-relevant enterprise milestone for ChatGPT. Enterprise Key Management customers are usually the organizations most sensitive to encryption, data control, and compliance boundaries.

Letting those workspaces use scheduled agents, Slack, apps, files, and MCP servers signals that ChatGPT is moving from assistant to governed workflow platform.

## Buyer take

If your organization already uses ChatGPT Enterprise with EKM, this is worth a controlled pilot. Start with a narrow workflow, a known data boundary, and clear owner approval before enabling Slack or custom MCP tools.

For buyers comparing ChatGPT Enterprise with Microsoft Copilot, ServiceNow, or Claude enterprise deployments, Workspace Agents should now be evaluated as a governed automation layer, not just a chat feature.

## What is still unclear

OpenAI has not published a full public list of eligible EKM workspaces, supported apps by region, or pricing changes tied specifically to Workspace Agents usage.
