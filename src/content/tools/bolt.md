# Bolt.new

Bolt.new is StackBlitz's browser-native AI app builder at [bolt.new](https://bolt.new). It lets users describe an idea, generate code, edit files, run the app, debug, manage Bolt Cloud resources, and publish without setting up a local development environment.

As of June 23, 2026, the old "frontend prototype only" description is not good enough. Current Bolt docs describe Bolt Cloud databases, hosting, custom-domain support for Pro users, JavaScript-based backend support, GitHub integration, Figma integration, Stripe integration, Supabase integration, and a model-choice workflow. That makes Bolt a real app-builder buyer option, not just a sandbox.

The buying question is whether you want the browser itself to be the build environment. If yes, Bolt is one of the strongest tools in the category. If you want a guided founder MVP workflow with backend and less code-workspace feel, start with [Lovable](/tools/lovable/). If you are already living in Vercel, Next.js, Tailwind, and shadcn, start with [v0](/tools/v0/).

## System Verdict

Pick Bolt.new if you want to build and iterate inside a browser tab. Its core advantage is the combination of AI prompting, full file-tree visibility, Code View, running previews, WebContainers, Bolt Cloud databases, built-in hosting, and direct project settings.

Bolt is especially useful for demos, classrooms, Chromebooks, locked-down work machines, hackathons, JavaScript web apps, and fast product experiments where local setup would slow everything down.

Do not pick Bolt because you think it eliminates engineering. It still needs review before production. Token usage can climb as the project grows. Bolt focuses on JavaScript-based web technologies, and its docs say mobile browsers are limited for Code View, project settings, database controls, and site visibility controls. That matters if your imagined workflow is building serious apps from a phone.

## Recent Developments

Serious coders comparing Bolt with Claude Code, Cursor, or local IDE workflows should test the same task in each tool before going annual.

## What Bolt Actually Is

Bolt is best understood as an AI-assisted browser development workspace. It is not a simple "generate a landing page" toy. The app can generate a project, expose the file tree, run the app, let you make code edits, show a preview, publish the project, and connect infrastructure pieces from the same browser environment.

The moat is StackBlitz WebContainers. WebContainers let the development environment run in the browser instead of requiring a local Node install. For schools, locked-down laptops, demos, and "I just need to try this now" workflows, that is a meaningful advantage.

The second moat is workflow compression. The same session can cover prompt-to-code, Code View, project settings, database settings, hosting, and deployment. The buyer is paying for speed and convenience, not just tokens.

## Pricing

Pricing was checked against [Bolt.new pricing](https://bolt.new/pricing) and [Bolt token docs](https://support.bolt.new/account-and-subscription/tokens) on 2026-06-23.

| Plan | Current public price | Buyer fit |

| Free | $0 | Best for testing Bolt, small demos, and occasional prototypes. Includes a 300K daily token limit and 1M tokens per month. |
| Pro | $25/month | Best first paid plan for solo builders. Starts at 10M tokens per month, removes the daily token cap, removes Bolt branding, adds custom domains, SEO boosting, expanded database capacity, and token rollover. |
| Teams | $30/member/month | Best for classrooms, agencies, and small teams that need centralized billing, access management, admin controls, private NPM registries, and design-system knowledge. |
| Enterprise | Custom | Best for larger companies that need advanced security, compliance, support, custom workflows, SLAs, data governance, retention policies, onboarding, and procurement support. |

The main pricing mistake is comparing Bolt only by plan sticker price. Bolt's docs say token usage is mostly tied to reading, understanding, and syncing project files. A larger project can use more tokens per message than a small project. That means the cheapest plan can be expensive in practice if you let a large app drift through long debugging loops.

Free is generous enough to test fit, but the 300K daily limit and 1M monthly limit are hard ceilings. Pro is the rational first paid plan once Bolt is part of your active workflow.

## Best Plan To Buy

Start with Free to test whether Bolt fits your actual building style. Use it for one small project, check how quickly token usage climbs, and see whether the browser environment feels like a productivity boost or a constraint.

Move to **Pro at $25/month** when you are actively building. The most important Pro gains are the 10M monthly starting token allocation, removal of the free daily token cap, custom domains, SEO boosting, expanded database capacity, and unused token rollover.

Use Teams when the purchase is about collaboration and administration, not just more AI usage. Teams is the better fit for classrooms, agencies, and organizations that need billing, access management, admin controls, private NPM registries, and design-system knowledge.

Enterprise is a procurement and governance plan. It is not necessary for solo builders unless a company requires SSO, compliance, custom workflows, SLAs, support, data governance, or procurement terms.

## Bolt Cloud And Backend Reality

Bolt is no longer accurately described as having no backend story. Current docs say Bolt Cloud can create and manage databases, expose tables, manage users and authentication-related settings, handle file storage, logs, secrets, security audit, server functions, and user management.

That said, Bolt is not identical to Lovable. Bolt's supported-technologies docs say it focuses on JavaScript-based web technologies and supports Node.js for backend work. PHP and Python backends are not compatible. Bolt also gives you the option to connect projects to Supabase for more advanced database management and configuration.

The practical buyer guidance is simple: use Bolt when you want the browser development workspace. Use Lovable when you want a more guided founder MVP experience. Use Cursor or Claude Code when a mature repo and engineering review are the main job.

## When To Pick Bolt.new

- **You want zero local setup.** Open the browser, generate, run, edit, and debug without installing Node.
- **You want code visibility.** Bolt is better than no-code tools when you want to see and edit the file tree.
- **You are teaching, demoing, or building on restricted machines.** WebContainers are a strong fit for Chromebooks, classrooms, and managed devices.
- **You need quick hosting.** Bolt hosting gives projects a free `.bolt.host` domain, while Pro users can connect custom domains.
- **You are building JavaScript web apps.** Bolt is strongest where the stack matches its supported technologies.

## When To Pick Something Else

- **Pick [Lovable](/tools/lovable/) for guided founder MVPs.** Lovable is usually better when a nontechnical founder wants a smoother prompt-to-product path.
- **Pick [v0](/tools/v0/) for Vercel-native React and Next.js work.** v0 is stronger when the final destination is Vercel, shadcn, Tailwind, GitHub, and PR review.
- **Pick [Cursor](/tools/cursor/) for mature repo work.** Bolt is fast for browser building, but existing production codebases need stronger local control and review.
- **Pick [Claude](/tools/claude/) or [Claude Code](/tools/claude-code/) when the task is multi-file reasoning in an existing repo.** Bolt's value is the browser workspace, not terminal-agent depth.
- **Pick another stack if you need PHP or Python backends.** Bolt's docs focus on JavaScript-based technologies.

## Token And Production Watch-Outs

- **Large projects burn more tokens.** Bolt says token use mostly comes from reading, understanding, and syncing project files.
- **Free has a daily cap.** Free users get 300K tokens per day and 1M tokens per month.
- **Rollover requires a paid subscription.** Paid subscription tokens can roll over for one additional month, but access depends on maintaining the paid subscription.
- **Reload pricing is plan-dependent.** Current token docs say reload cost varies by plan and is shown in-app, so AiPedia should not publish an unsupported fixed reload price.
- **Mobile browser support is limited.** Bolt works best in Chrome and other Chromium-based desktop browsers. Mobile browsers can start projects, chat, and publish, but Code View, project settings, database controls, and site visibility controls require desktop.
- **Browser runtime limits still exist.** WebContainers are powerful, but large dependencies and heavy builds may still be slower than a local dev environment.
- **Production still needs engineering review.** Check auth, database access, secrets, deployment settings, logging, dependency health, and generated code quality before real users arrive.

## FAQ

**Is Bolt.new free?**

Yes. The Free plan includes public and private projects, a 300K daily token limit, 1M tokens per month, Bolt branding, website hosting, and unlimited databases as verified on 2026-06-23.

**What Bolt.new plan should I buy first?**

Most solo builders should start free, then move to Pro at $25/month once the free token caps slow active work.

**Does Bolt.new include databases?**

Yes. Current docs say Bolt Cloud includes built-in database creation and management. Bolt also offers a Supabase connection path for more advanced database management and configuration.

**Does Bolt.new run locally?**

No. Bolt runs in the browser using StackBlitz WebContainers. That is the point: the development environment does not require local setup.

**Does Bolt.new support Python or PHP backends?**

No. Current supported-technology docs say Bolt focuses on JavaScript-based web technologies, supports Node.js for backend work, and does not support PHP or Python backends.

**Bolt.new vs Lovable: which is better?**

Bolt is better for browser-native build/run/debug workflows. Lovable is usually better for nontechnical founder MVPs that need more guided app-builder flow.

**Bolt.new vs v0: which is better?**

Bolt is broader as a browser development workspace. v0 is better when the buyer wants Vercel-native React, Next.js, Tailwind, shadcn, deployment, and PR workflows.
